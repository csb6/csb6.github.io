Bthreads (August 2019)

Repository

A Python library supporting behavioral programming (BP), an experimental programming paradigm focused on creating separate behaviors or "b-threads" to be run in parallel or as coroutines. My implementation uses Python generators as a sort of coroutine.

The aim of BP is to write code incrementally, with each b-thread waiting for, blocking, or requesting events to occur. Since all actions the b-threads take have to be requested before they can happen, they are much easier to reason about than dense, interdependent code. Simply by looking at a log of events, developers can easily develop in an "append-only" style, adding new behaviors to change the sequence of events that are triggered without knowing or caring about the code in prior behaviors.

See the repository README for a more in-depth explanation with examples.

  • Language/Technology: Python
  • Write b-threads like normal Python generators
  • Repository has example of fairly complex 2-player tic-tac-toe program which validates each player's moves and checks for winning conditions.

Mail.py (May-June 2019)

Repository

A bare-bones IMAP email client in Tkinter, still not feature-complete. Right now, it supports loading/viewing the messages in an email account's inbox and sending plaintext messages from the same account.

The purpose of this project was to see how far I could push my Python skillset, and while I managed to get the very basic features implemented, parsing messages, which come in a variety of HTML and plaintext formats, proved difficult. Interacting with the IMAP protocol also took more effort than I expected. However, the client still works as a simple email inbox viewer and message sender.

  • Language/Technology: Python, Tk, IMAP, SQLite
  • Load your inbox into a local SQL database
  • Model-View-Adapter Architecture
  • Send emails
  • See senders, subjects, time of sending for all messages

Event Simulator (May 2019)

Repository

A simulation module based on Markov chains. Custom simulations can be created/run by specifying states/transitions in a JSON file and Python code in a second file to be executed after each event.

This was a quick project, but a fascinating way to apply Markov chains to something beyond making text. I also experimented with passing functions as arguments and organizing the program in an object-oriented way.

  • Language/Technology: Python
  • Create an unlimited number of states/transitions
  • Assign probabilities that each transition occurs
  • Structured as a reusable Python module

Squad Tactical Game (May-July 2018)

Repository

A 2D, top-down, turn-based tactical game based on the X-COM series. The code is a bit messy because I experimented with inheritance and later a component-based structure for the game's entity objects, but most features are implemented.

The most notable portion of the project is my implementation of the A* pathfinding algorithm, which allows the soldiers to move to a clicked location, avoiding any obstacles along the way. One of my most fun (and challenging) projects.

  • Language/Technology: Ruby, Tk
  • Click to move: soldiers will pathfind to selected location.
  • Chance to hit/damage varies with distance, nearby cover
  • Simple CSV format for level creation
  • Simple enemy AI that flanks your soldiers
